About

I recently earned my Computer Science degree from Boise State University. The parts of the field I keep coming back to are the ones close to the machine: compilers, memory management, and the security boundaries that keep software trustworthy.

My coursework and personal projects span both sides of that interest. On the development side I've built a Huffman file compressor in Zig, designed and implemented an interpreted programming language in Java, and shipped full-stack web applications. On the security side I focus on network and systems security and secure coding, which really come down to thinking about how software fails and how to keep it from failing.

AI tools are a real part of how I work. I use them to move faster on the routine parts of a project, like scaffolding code, talking through design tradeoffs, debugging tricky errors, and filling out tests, so I can spend more of my time on the hard problems. I always treat what they produce as a starting point I read, understand, and verify rather than something to paste in blindly.

I'm now looking for full-time roles where I can keep doing both: writing solid software and helping protect it.

Zig Huffman Encoder

2024

A command-line file compression tool written in Zig that implements Huffman encoding from scratch. Builds frequency tables with hashmaps, constructs the Huffman tree, and performs bit-level packing with custom memory allocators. Building it pushed me deep into manual memory management and low-level I/O.
